WindHex Mobile: Table Editor

Using the table file editor

Creating a table file is fairly simple. The only requirements are that the table entries (1) use the left to right format and (2) have even numbered hex entries as seen below.
Example:
41=A
42=B
0141=あ
80=an
81=hello
82=my friend
FE=[newl]
FF=[end]
1=A and 234=happy ; Invalid entries.

Main Menu Bar
  • On the menu bar, clicking the "Arrow (Return)" icon on the left side allows you to return to the hex editor window.
  • The option menu selections are self explanatory.

Depending on the button used to initiate the table editor, you will have access to certain button functions. One of these being the "Table Undo" button on the "Table Editor's" menu bar. This feature is very limited as it only works on the "New File" initiated process from the hex window's menu bar. More support might be added in the future.

If the table editor is initiated using the "Edit Table" button on the hex window's menu bar, then the undo feature is disabled. Only when using the "Edit Table File" function can you load a shift-jis file and convert it to UTF format.

At the lower half of the screen, there is a drop down list that gives you the ability to select a character set from a list of unicode tables. If the "not valid" image (Circle with a line) appears, it means the selected unicode table is not supported on the current system.
